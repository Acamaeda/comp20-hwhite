Security thing

Introduction

The product in question is a web server and database that stores scores for the game 2048 and retrieve high scores of several types.

Methodology

The first method used was to use curl to submit problematic data.

Second, I tried to flood the page with requests, but it didn't work because it almost crashed my VM.

I also attempted to use Burp but the scan feature is paid only and I'm too tired to figure out the rest.

Abstract of findings

Hackers can insert hostile scripts into the webpages through the submissions.

Issues found

CORS Issue

Moderate

I submitted illegitimate scores with curl.

Your CORS is set up to accept requests from all sites. You should set it to only accept requests from the site that should be accessing it.

Unsanitized data submissions.

High severity, anyone can easily insert harmful code and it is a common vulnerability.

I submitted a score with a script to set the page background to yellow, and the scores page background is now yellow

This can be resolved by sanatizing your inputs, replacing \< and \> with \< and \> respectively. Further attacks are possible without these symbols via "jsfuck", so look into that as well.
